Instructions for creating `-brief` summary files from older monthly or yearly files.

When creating a brief file (for example `Daily-2026-03-brief.md` from `Daily-2026-03.md`):

1. Read the original file fully.
2. Preserve key facts, decisions, emotions, conflicts, and turning points.
3. Keep names, dates, and numbers when they matter.
4. Drop repetitive mundane details unless they show a pattern worth noting.
5. Keep the user's voice and perspective.
6. Output only the summary content, in the same Markdown format as the original file.
